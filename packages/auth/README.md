# @captable/auth

Better Auth implementation for Captable with Next.js integration, including extended session data with member information.

## Features

- **Email/password authentication** with Better Auth
- **Google OAuth** integration
- **Next.js App Router** support with server actions
- **Extended session** with member data:
  - `isOnboarded`: boolean
  - `companyId`: string  
  - `memberId`: string
  - `companyPublicId`: string
  - `status`: MemberStatusEnum | ""
- **Automatic cookie handling** for server actions
- **Middleware integration** for route protection

## Setup

### 1. API Route

The Better Auth API route is automatically configured at `/api/auth/[...all]` and includes:
- Sign in/out endpoints
- Session management
- OAuth flows

### 2. Environment Variables

Required environment variables:
```bash
BETTER_AUTH_SECRET="xxx" # Generated by `openssl rand -base64 32`
BETTER_AUTH_URL="https://your-domain.com"
NEXT_PUBLIC_BASE_URL="https://your-domain.com"

# Optional: Google OAuth
GOOGLE_CLIENT_ID="your-google-client-id"
GOOGLE_CLIENT_SECRET="your-google-client-secret"
```

## Usage

### Server-side (RSC & Server Actions)

```typescript
import { serverSideSession } from "@captable/auth/server";
import { headers } from "next/headers";

export default async function Page() {
  const session = await serverSideSession({ headers: await headers() });
  
  if (!session) {
    return <div>Not authenticated</div>;
  }
  
  console.log(session.user.isOnboarded);
  console.log(session.user.companyId);
  console.log(session.user.memberId);
  console.log(session.user.status);
  
  return <div>Hello {session.user.name}</div>;
}
```

### Server Actions

```typescript
import { signInEmailAction, signUpEmailAction, signOutAction } from "@captable/auth";

async function handleSignIn(formData: FormData) {
  "use server";
  
  const email = formData.get("email") as string;
  const password = formData.get("password") as string;
  
  try {
    await signInEmailAction(email, password);
    // Cookies are automatically set via nextCookies plugin
    redirect("/dashboard");
  } catch (error) {
    // Handle error
  }
}
```

### API Routes

```typescript
import { serverSideSession } from "@captable/auth/server";

export async function GET(request: Request) {
  try {
    const session = await serverSideSession({ headers: request.headers });
    
    // Access member data
    const { isOnboarded, companyId, memberId, status } = session.user;
    
    return Response.json({ success: true, user: session.user });
  } catch (error) {
    return Response.json({ error: "Unauthorized" }, { status: 401 });
  }
}
```

### Usage Patterns

**Server Components & Server Actions:**
```typescript
import { serverSideSession } from "@captable/auth/server";
import { headers } from "next/headers";

const session = await serverSideSession({ headers: await headers() });
```

**API Routes:**
```typescript
import { serverSideSession } from "@captable/auth/server";

export async function GET(request: Request) {
  const session = await serverSideSession({ headers: request.headers });
}
```

**Middleware (performance optimized):**
```typescript
import { getSessionCookie } from "better-auth/cookies";

// Use cookie checking instead of full session for performance
const sessionCookie = getSessionCookie(request);
```

### Client-side (React Components)

```typescript
import { clientSideSession, signIn, signOut } from "@captable/auth";

export function MyComponent() {
  const { data: session, isPending } = clientSideSession();
  
  if (isPending) return <div>Loading...</div>;
  
  if (!session) {
    return <button onClick={() => signIn.email({ email, password })}>Sign In</button>;
  }
  
  return (
    <div>
      <p>Welcome {session.user.name}</p>
      <p>Company: {session.user.companyId}</p>
      <p>Status: {session.user.status}</p>
      <button onClick={() => signOut()}>Sign Out</button>
    </div>
  );
}
```

### Middleware

The middleware automatically protects routes using Better Auth session cookies:

```typescript
// Automatically protects these routes
const protectedRoutes = ["/dashboard", "/onboarding", "/company"];

// Redirects authenticated users from these routes
const authRoutes = ["/login", "/signup"];
```

## Session Data

The extended session includes all member information:

```typescript
interface ExtendedSession {
  user: {
    id: string;
    name: string;
    email: string;
    image?: string;
    emailVerified: boolean;
    createdAt: Date;
    updatedAt: Date;
    // Extended member data
    isOnboarded: boolean;
    companyId: string;
    memberId: string;
    companyPublicId: string;
    status: MemberStatusEnum | "";
  };
  session: {
    id: string;
    expiresAt: Date;
    token: string;
    ipAddress?: string | null;
    userAgent?: string | null;
    userId: string;
    activeOrganizationId?: string | null;
  };
}
```

## Key Features

- **Single Function**: One `serverSideSession({ headers })` function for all server-side usage
- **Extended Member Data**: Automatically includes the most recently accessed active member
- **Automatic Cookie Handling**: The `nextCookies` plugin handles cookie setting in server actions
- **Route Protection**: Middleware automatically protects routes using session cookies
- **Type Safety**: Full TypeScript support with proper type inference
- **Performance**: Cookie-only checks in middleware for fast route protection

For more information, see the [Better Auth documentation](https://www.better-auth.com/docs/integrations/next).
